Die etablierten Projektmanagements-Strategien ([[Agile\|Agile]], [[Scrum\|Scrum]], [[Waterfall\|Waterfall]] etc.) gehen davon aus, dass es langsam und teuer ist, Code zu schreiben. Deshalb definieren sie viel Zeremonie, welche sicherstellen soll, dass auf keinen Fall Code geschrieben wird, welcher nicht gebraucht wird. Tatsächlich ist es heute aber so einfach (und billig) wie nie, in kürzester Zeit einen Prototypen zu entwickeln, auf welchem dann iteriert werden kann[^1].
